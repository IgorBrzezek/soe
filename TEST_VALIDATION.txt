VALID COMMAND EXAMPLES (for actual testing):
=============================================

1. Start simulator on Windows serial port COM1:
   python serial_device_0.0.1.py --comport COM1 --hostname myrouter

2. Start simulator on Linux serial port:
   python serial_device_0.0.1.py --comport /dev/ttyUSB0 --hostname myrouter

3. Start simulator on Windows named pipe (requires pywin32):
   python serial_device_0.0.1.py --namedpipe mydevice --hostname myrouter

4. Custom baud rate and line format:
   python serial_device_0.0.1.py --comport COM1 --baud 115200 --line 8N2N --hostname router

5. With debugging output:
   python serial_device_0.0.1.py --comport COM1 --debug

6. Load custom commands from file:
   python serial_device_0.0.1.py --comport COM1 --cmdfile commands.txt

7. With custom banner message:
   python serial_device_0.0.1.py --comport COM1 --login-message "Welcome to my device"


All validation happens BEFORE attempting any connection.
Invalid options cause immediate exit with exit code 1.
No silent failures - all errors are clearly reported.
